# План внедрения нового провайдера для llama.cpp

## 1. Получение списка моделей и параметров

### Стандартные endpointы
- Для Ollama: `/api/tags` и `/api/ps` для получения моделей и статуса
- Для llama.cpp: вероятно, аналогичные endpointы или специфичные для llama.cpp API

### Структура данных модели
- Название модели
- Размер модели (в байтах/GB)
- Размер контекста (tokens)
- Статус (загружена/доступна)
- Тип модели (embedding/vision/text)

### Решение
Создать общий интерфейс `ModelInfo` с полями:
```javascript
{
  name: string,
  size: number,
  contextLength: number,
  status: 'running' | 'available',
  type: 'text' | 'vision' | 'embedding'
}
```

Каждый провайдер реализует метод `getModels()` который возвращает массив таких объектов.

## 2. Архитектура модулей

### Файловая структура
```
src/
├── providers/                 # Провайдеры
│   ├── BaseProvider.js        # Базовый класс
│   ├── OllamaProvider.js      # Реализация для Ollama
│   └── LlamaCppProvider.js   # Реализация для llama.cpp
├── api/                      # API обработчики
│   ├── models.js             # Работа с моделями
│   ├── chat.js               # Работа с чатом
│   └── show.js               # Получение информации о модели
├── config/                   # Конфигурация
│   └── providers.js          # Загрузка конфигурации провайдеров
├── utils/                    # Утилиты
│   └── fileProcessor.js     # Обработка файлов
└── server.js                 # Главный сервер
```

### Обновленный server.js
Вынести логику в отдельные модули:
- `src/api/models.js` - обработка `/api/models`
- `src/api/chat.js` - обработка `/api/chat` 
- `src/api/show.js` - обработка `/api/show`
- `src/providers/` - реализация провайдеров
- `src/config/providers.js` - загрузка конфигурации
